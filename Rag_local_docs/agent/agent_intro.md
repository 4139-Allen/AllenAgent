# AI Agent 智能体基础

## 什么是智能体

智能体（AI Agent）是一种能够自主感知环境、做出决策并执行动作的人工智能系统。与传统的 LLM 不同，智能体具有以下核心能力：

1. **感知**：通过工具获取外部信息（搜索、数据库、API）
2. **规划**：将复杂任务分解为可执行的步骤
3. **执行**：调用工具完成具体动作
4. **记忆**：保持短期对话历史和长期知识存储

## 核心循环：ReAct

智能体的核心循环是 **ReAct（Reasoning + Acting）**：

```
思考（Thought）→ 行动（Action）→ 观察（Observation）→ 循环
```

### 工作流程

1. **Thought**：分析当前状态，决定下一步行动
2. **Action**：调用工具执行具体操作
3. **Observation**：获取工具返回的结果
4. **循环**：根据观察结果继续思考，直到任务完成

### 示例

```
用户：帮我查一下北京今天的天气

Thought: 我需要查询北京今天的天气信息，可以使用天气API
Action: call_weather_api(city="北京", date="today")
Observation: 北京今天晴，气温15-25℃，空气质量良好
Thought: 已经获取到天气信息，可以回复用户了
Answer: 北京今天天气晴朗，气温15-25℃，空气质量良好。
```

## 智能体架构

### 单智能体架构

最简单的形式，一个 LLM 配合工具集：

```
┌─────────────────────────────────────┐
│            AI Agent                 │
│  ┌─────────┐  ┌─────────────────┐  │
│  │   LLM   │  │  Tools/Actions  │  │
│  │ (大脑)   │  │  (手和脚)       │  │
│  └─────────┘  └─────────────────┘  │
└─────────────────────────────────────┘
```

### 多智能体协作

多个专业智能体协同工作：

```
┌─────────────┐     ┌─────────────┐
│  Planner    │────→│  Coder      │
│  (规划者)    │     │  (编码者)    │
└─────────────┘     └─────────────┘
       │                   │
       ▼                   ▼
┌─────────────┐     ┌─────────────┐
│  Reviewer   │←────│  Tester     │
│  (审查者)    │     │  (测试者)    │
└─────────────┘     └─────────────┘
```

## 常见工具类型

### 1. 信息获取类
- **搜索引擎**：获取实时网络信息
- **数据库查询**：检索结构化数据
- **API调用**：获取第三方服务数据

### 2. 代码执行类
- **Python解释器**：执行计算、数据处理
- **Shell命令**：系统操作、文件管理

### 3. 文件操作类
- **读写文件**：持久化存储
- **文档解析**：处理PDF、Word等格式

## 主流 Agent 框架

| 框架 | 特点 | 适用场景 |
|------|------|----------|
| LangChain | 生态丰富，组件多 | 通用场景 |
| LlamaIndex | 专注 RAG | 知识库问答 |

| CrewAI | 多智能体协作 | 复杂工作流 |

## 实践建议

1. **从小开始**：先实现单工具调用，再逐步扩展
2. **明确边界**：定义清楚智能体能做什么、不能做什么
3. **错误处理**：工具调用可能失败，需要降级策略
4. **人机协作**：关键决策点加入人工确认
5. **日志追踪**：记录每步思考和行动，便于调试
